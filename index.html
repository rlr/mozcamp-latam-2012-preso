Our typical stack

  • HTML/CSS/JS (Open Web Technologies!)
  • Python+Django
  • Celery
  • MySQL, Memcached, Redis, ElasticSearch

playdoh

  • github.com/mozilla/playdoh
  • Project used for bootstrapping new sites
  • Builds in our conventions and best practices
    • Jinja2 templates
    • (More) Secure by default
    • L10n enhancements
  • Documentation: playdoh.readthedocs.org

Shared libraries

  • We maintain and contribute to lots of open source libraries.
  • Useful to sites outside of Mozilla
  • Use them, report issues, contribute fixes
  • bleach
    HTML sanitization of user input
  • django cache machine
    Automatic caching and invalidation for Django models
  • django-cronjobs
    A simple cron-running management command for Django
  • waffle
    A feature flipper for Django
  • many, many more

The dev process

Choose a bug (Bugzilla)

Github: Fork the repo (Github)

Implement fix in a branch

Pull request (Github)

Code review (Github)

Merge to master (Github)

Run test suite (Jenkins)

Deploy to dev site (Automated scripts)

Deploy to stage and prod site (push a button)

Why contribute?

  • Cool, interesting, hard and FUN problems
